01
Descripción general
¿Qué es este proyecto?

Este proyecto consiste en el diseño, despliegue y documentación completa de la infraestructura tecnológica de una agencia real de orientación y admisión de estudiantes internacionales: Espanito.

Se trata de un proyecto intermodular de 1º de ASIR que integra todos los módulos del curso en torno a un caso de uso concreto y funcional: una empresa que existe de verdad y que el propio autor dirige. Esto convierte el proyecto en algo más que un ejercicio académico — es la documentación técnica de una infraestructura real.

Espanito es una agencia de orientación académica internacional con sede en Madrid, especializada en ayudar a estudiantes iraníes a acceder a universidades en España y Europa. La infraestructura IT diseñada en este proyecto da soporte tecnológico completo a todas las operaciones de la agencia.

El objetivo del proyecto no es instalar sistemas de forma aislada, sino diseñar y articular una infraestructura coherente: red, servidores, base de datos, gestión de datos estructurados y arquitectura cloud, todo documentado y organizado en un repositorio profesional en GitHub.

02
Justificación del proyecto
El problema que resuelve

Espanito gestiona de forma manual un conjunto de procesos complejos: seguimiento de expedientes de estudiantes, control documental, coordinación entre asesores, agenda de citas y facturación. Sin una infraestructura IT estructurada, estos procesos son lentos, propensos a errores y difíciles de escalar.

Gestión desorganizada

Información de estudiantes, documentos y solicitudes dispersa en correos y hojas de cálculo sin estructura centralizada.

Falta de control

Sin seguimiento estructurado del estado de cada candidatura es imposible gestionar eficientemente múltiples estudiantes.

Comunicación ineficiente

La coordinación entre asesores y estudiantes carece de un sistema centralizado de citas y notificaciones.

La solución

Una infraestructura IT completa: red local segura, servidor con base de datos relacional, servicios DNS/DHCP, datos estructurados y acceso cloud gestionado.

La infraestructura diseñada en este proyecto da respuesta directa a cada uno de estos problemas, aplicando los conocimientos adquiridos en todos los módulos de 1º de ASIR.

03
Usuarios del sistema
¿Para quién está pensado?

El sistema da servicio a dos tipos de usuarios con necesidades y niveles de acceso diferenciados:

A

Asesores y equipo de la agencia

Acceden a la base de datos para gestionar expedientes de estudiantes, documentos, citas y pagos. Necesitan conexión segura a la red interna y acceso a los servicios del servidor.

D

Dirección (administrador)

Acceso completo al sistema: administración del servidor, gestión de usuarios de red y base de datos, backups y monitorización general de la infraestructura.

Desde el punto de vista académico, este proyecto demuestra la capacidad de diseñar, implantar y documentar una infraestructura IT real, integrando los conocimientos de todos los módulos de 1º de ASIR en torno a un caso de uso coherente y funcional.

Estructura del proyecto
Módulos del proyecto

El proyecto se divide en seis bloques técnicos, cada uno correspondiente a un módulo del curso y aportando una capa diferente a la infraestructura total:

0370

Planificación y administración de redes

Diseño de la topología de red en estrella, plan de direccionamiento IP (192.168.10.0/24), configuración de servicios DNS y DHCP, dispositivos: router, switch gestionable y puntos de acceso WiFi.

0369

Implantación de sistemas operativos

Instalación y configuración de Ubuntu Server 22.04 LTS (servidor principal) y Windows 11 (equipos de los asesores). Gestión de usuarios, grupos, permisos y servicios básicos: SSH, compartición de archivos con Samba.

0372

Gestión de bases de datos

Base de datos relacional MySQL con 8 tablas: estudiantes, asesores, universidades, programas, solicitudes, documentos, citas y pagos. Incluye consultas SQL, gestión de usuarios con permisos diferenciados y scripts de backup.

0373

Lenguajes de marcas

Documento XML para exportación estructurada de datos de estudiantes y sus solicitudes, con esquema XSD completo para validación. Integrado como módulo de exportación de la base de datos.

MPO

Computación en la nube

Diseño de la arquitectura equivalente en AWS: EC2 para el servidor de aplicación, RDS para la base de datos gestionada y S3 para almacenamiento de documentos. Estimación de costes mensuales con calculadora AWS.

1709

Itinerario personal para la empleabilidad I

Perfil profesional, investigación de empresas del sector (Telefónica Tech, Indra, NTT DATA), análisis de perfiles profesionales en LinkedIn y GitHub, portfolio y reflexión personal.

06
Conclusiones técnicas
Aprendizajes clave

Desarrollar este proyecto sobre una empresa real ha permitido conectar los conocimientos del curso de una forma que un caso ficticio no habría conseguido:

01

La red y el sistema operativo son inseparables

Configurar DNS en Ubuntu Server para que funcione con los clientes Windows de la red demostró que ambas capas deben diseñarse juntas, no como módulos independientes.

02

El diseño de datos impacta en todo lo demás

El esquema relacional de la base de datos determinó cómo se estructuran los XML de exportación, cómo acceden los asesores al sistema y qué consultas son posibles. Un modelo de datos deficiente afecta a toda la infraestructura.

03

La documentación es parte del trabajo técnico

Documentar cada decisión — por qué este sistema operativo, por qué este rango IP, por qué esta estructura de tablas — hizo el razonamiento técnico más riguroso y el resultado más profesional.

04

Los proyectos reales tienen restricciones que los ficticios no tienen

Al basarse en Espanito, el diseño tuvo que adaptarse a necesidades concretas: número real de usuarios, tipos de documentos reales, flujos de trabajo existentes. Esto es infinitamente más formativo que un caso inventado.
